教学地图

从基础认知到高级体验优化，按学习曲线逐步展开。

基础 进阶 综合实践
基础 01

视觉层级与可读性

让用户一眼知道“先看哪里、再做什么”，是好界面的起点。

  • 用字号、留白、色彩和对比建立主次。
  • 标题负责吸引，正文负责解释，按钮负责行动。
  • 实现上优先设计排版系统和间距规则，而不是先堆组件。
基础 02

可供性与操作暗示

元素要“看起来就像能点、能拖、能切换”，这会显著降低学习成本。

  • 按钮形状、阴影、光标和按压状态都在暗示操作方式。
  • 别让纯文本伪装成交互控件。
  • 实现时至少准备默认、悬停、按下、禁用四种状态。
基础 03

即时反馈与系统状态

点击之后的微反馈，决定用户是否觉得系统可靠、流畅、可控。

  • 微动效、颜色变化、进度条和提示文案都属于反馈。
  • 加载、成功、失败不能混在一起表达。
  • 实现上用类名切换或状态机管理反馈最清晰。
进阶 01

渐进披露与认知减负

不是所有信息都要一次性塞给用户，分阶段呈现更符合认知规律。

  • 先展示关键行动，再按需展开解释和高级选项。
  • 折叠面板、分步向导、抽屉层都属于渐进披露。
  • 实现时注意展开前后布局和焦点管理。
进阶 02

表单体验与错误预防

好的表单不是“出错后批评用户”，而是提前帮助用户成功完成任务。

  • 字段标签清楚、输入状态明确、错误提示具体。
  • 错误信息应该说明问题和修正方法。
  • 实现上把校验拆成实时提醒和提交校验两层。
综合实践

一致性、动效与无障碍

高级体验来自多个维度共同工作，而不是某一个炫酷组件。

  • 一致的颜色、圆角、间距和反馈机制会建立产品可信度。
  • 动效要服务理解，而不是制造干扰。
  • 实现时加入键盘可达性、语义标签和对比度意识。

01. HTML 语法入门

HTML 可以理解为网页的骨架。标签负责告诉浏览器：哪里是标题，哪里是段落，哪里是按钮。

<section>
  <h1>我的第一个卡片</h1>
  <p>这是对卡片内容的说明。</p>
  <button>点我</button>
</section>
你要记住的重点

`<h1>` 是标题，`<p>` 是段落，`<button>` 是按钮。网页先有结构，后有美化。

02. 按钮怎么写得更好看

按钮的核心通常来自四部分：背景色、文字颜色、内边距、圆角。再进一步，可以加上悬停和阴影。

button {
  background: #d6613b;
  color: white;
  padding: 12px 16px;
  border: none;
  border-radius: 14px;
}
你要记住的重点

`padding` 决定按钮“厚不厚”，`border-radius` 决定圆润程度，颜色决定情绪和强调关系。

03. 阴影为什么能增强高级感

阴影不是为了“炫”，而是为了告诉用户：这个元素悬浮在页面上，有层级、有重点、更值得注意。

.card {
  background: white;
  border-radius: 22px;
  box-shadow: 0 18px 34px rgba(0,0,0,0.14);
}
你要记住的重点

`box-shadow` 数值越大，阴影越扩散；透明度越低，阴影越柔和。轻阴影适合普通卡片，重阴影适合重点内容。

04. CSS 选择器怎么理解

选择器就是“我要修改谁”。你写样式之前，先要学会准确选中页面元素。

p {
  color: #586173;
}

.highlight {
  color: #d6613b;
}
你要记住的重点

`p` 会选中所有段落，`.highlight` 会选中 class 为 `highlight` 的元素。类选择器是做组件样式最常用的方法。
